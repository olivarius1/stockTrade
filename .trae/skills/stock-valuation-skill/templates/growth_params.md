# 8种估值模型 -- 完整参数参考表

本文档为 `build_report.py` 支持的全部 8 种估值模型提供参数参考，包括因子权重、适用行业、PE/PB 区间建议、新增因子说明及缺失处理规则。

---

## 一、模型总览与适用场景

| 模型标识 | 模型名称 | 适用行业 | 核心特征 |
|----------|----------|----------|----------|
| `staples` | 必选消费 | 食品饮料、乳制品、调味品、文具、日用品 | 业绩稳定、现金流好、防御性强、需求刚性 |
| `discretionary` | 可选消费 | 免税旅游、白酒、家电、服装、珠宝 | 品牌溢价明显、受消费周期影响、PEG敏感 |
| `tech` | 科技制造 | 半导体、汽车零部件、湿电子化学品、软件 | 高研发投入、增速快、PEG权重最高 |
| `cyclical` | 周期资源 | 有色金属、航运、化工、煤炭、钢铁 | 业绩随商品价格大幅波动、需追踪商品价格位置 |
| `soe` | 央企基建 | 建筑、基建、电力、交运、电信 | 股息率高、订单驱动、PB估值为主 |
| `bank` | 银行保险 | 银行、保险、券商 | PB为核心、ROE和不良率决定估值、PE权重为0 |
| `realestate` | 地产 | 房地产开发、物业管理 | NAV折价为核心、去化率和杠杆率为关键 |
| `pharma` | 医药消费 | 化学制药、中药、医疗器械、医疗服务 | 营收增速权重高、受政策周期影响大 |

### 模型选择指南

1. **业绩稳定增长、现金流好** --> `staples` 或 `pharma`
2. **品牌驱动、PEG敏感** --> `discretionary`
3. **高研发、高增速** --> `tech`
4. **随商品价格/宏观经济波动** --> `cyclical`
5. **高分红、央国企、订单驱动** --> `soe`
6. **金融行业（银行/保险）** --> `bank`
7. **房地产开发** --> `realestate`
8. **医药/医疗** --> `pharma`

> 旧版 `growth` 模型已自动映射为 `staples`，保持向后兼容。

## 四、新增因子说明与输入方式

### 4.1 因子分类

全部因子分为两类：

**基础因子**（由K线数据和必要参数自动计算，无需额外输入）：

| 因子名 | 代码键 | 计算来源 |
|--------|--------|----------|
| PE(TTM) | `pe` | 腾讯API qt[39] + K线收盘价反推 |
| PB | `pb` | 腾讯API qt[46] + K线收盘价反推 |
| PEG | `peg` | PE(TTM) / (预期增速 * 100) |
| MA偏离度 | `ma` | 收盘价偏离MA20/MA60的百分比 |
| 量能 | `vol` | 当日成交量 / 20日平均成交量 |
| 波动率 | `vola` | 当日振幅 = (最高-最低) / 收盘价 |

**可选因子**（需要通过命令行 `--键:值` 格式额外输入）：

| 因子名 | 代码键 | 输入格式 | 单位 | 说明 |
|--------|--------|----------|------|------|
| 商品价格偏离 | `commodity_dev` | `--commodity_dev:-0.15` | 小数 | 当前商品价格相对历史均值的偏离度，负值=低于均值（低估信号）。例如铜价历史均值7万，当前5.95万，输入 `-0.15` |
| 产能利用率 | `capacity_util` | `--capacity_util:0.85` | 小数(0-1) | 当前产能利用率百分比。例如85%输入 `0.85` |
| ROE | `roe` | `--roe:0.15` | 小数 | 最近一年净资产收益率。例如15%输入 `0.15` |
| 股息率 | `dividend_yield` | `--dividend_yield:0.05` | 小数 | 最近年度股息率。例如5%输入 `0.05` |
| 不良/偿付 | `npl_ratio` | `--npl_ratio:0.012` | 小数 | 不良贷款率。例如1.2%输入 `0.012` |
| NAV折价 | `nav_discount` | `--nav_discount:0.7` | 小数 | 股价/每股NAV的比值。低于1为折价。输入 `0.7` 表示折价30% |
| 去化率 | `clearance_rate` | `--clearance_rate:0.65` | 小数(0-1) | 当前在建项目去化率。例如65%输入 `0.65` |
| 杠杆率 | `leverage` | `--leverage:0.55` | 小数(0-1) | 有息负债率。例如55%输入 `0.55` |
| 研发费用率 | `rd_ratio` | `--rd_ratio:0.08` | 小数 | 研发费用/营业收入。例如8%输入 `0.08` |
| 毛利率稳定性 | `margin_stability` | `--margin_stability:0.02` | 小数 | 近3年毛利率的标准差。越小表示越稳定。输入 `0.02` 表示标准差2个百分点 |
| 品牌溢价度 | `brand_premium` | `--brand_premium:1.8` | 小数 | 公司毛利率 / 行业平均毛利率。输入 `1.8` 表示毛利率是行业均值的1.8倍 |
| 订单增速 | `order_growth` | `--order_growth:0.20` | 小数 | 新签合同/订单同比增速。例如20%输入 `0.20` |
| 营收增速 | `revenue_growth` | `--revenue_growth:0.18` | 小数 | 最新营收同比增长率。例如18%输入 `0.18` |

## 五、可选因子缺失时的处理规则

### 5.1 处理逻辑

当某个可选因子未被用户输入（缺失）时，系统按以下规则处理：

1. **识别缺失因子**：遍历模型权重组，找出用户未提供值（`None`）且权重 > 0 的可选因子
2. **计算缺失权重总和**：将所有缺失因子的权重累加，记为 `missing_weight_sum`
3. **按比例重分配**：将 `missing_weight_sum` 按比例分配到已有值的因子上
4. **归一化**：重新归一化所有活跃因子权重，确保总和 = 1（即100%）

### 5.2 重分配公式

```
对于每个有值的因子 k：
  新权重 = 原权重 + 原权重 / 活跃因子权重总和 * 缺失权重总和

归一化后：
  最终权重 = 新权重 / 所有新权重之和
```

### 5.3 示例

**bank 模型全部因子齐全时：**
```
PB(30%) + ROE(25%) + 股息率(15%) + 不良/偿付(12%) + MA偏离(10%) + 波动率(8%) = 100%
```

**bank 模型缺失 `roe` 和 `npl_ratio` 时：**
```
缺失权重 = 25% + 12% = 37%
活跃因子原权重 = 30% + 15% + 10% + 8% = 63%

重分配后：
  PB = 30% + 30%/63% * 37% = 47.6%
  股息率 = 15% + 15%/63% * 37% = 23.8%
  MA偏离 = 10% + 10%/63% * 37% = 15.9%
  波动率 = 8% + 8%/63% * 37% = 12.7%

归一化后（总和已是100%）：
  PB(48%) + 股息率(24%) + MA偏离(16%) + 波动率(12%) = 100%
```

**cyclical 模型缺失 `commodity_dev` 和 `capacity_util` 时：**
```
缺失权重 = 20% + 8% = 28%
活跃因子原权重 = 25% + 12% + 15% + 10% + 10% = 72%

重分配后：
  PE = 25% + 25%/72% * 28% = 34.7%
  PB = 12% + 12%/72% * 28% = 16.7%
  MA偏离 = 15% + 15%/72% * 28% = 20.8%
  量能 = 10% + 10%/72% * 28% = 13.9%
  波动率 = 10% + 10%/72% * 28% = 13.9%

归一化后：
  PE(35%) + PB(17%) + MA偏离(21%) + 量能(14%) + 波动率(14%) = 100%
```

### 5.4 影响评估

| 缺失因子数量 | 对评分精度的影响 | 建议措施 |
|-------------|-----------------|----------|
| 0个缺失 | 最精确 | 完整使用所有因子 |
| 1-2个缺失 | 轻微影响，偏差约5-10分 | 核心因子（高权重）齐全即可接受 |
| 3个及以上缺失 | 中等影响，偏差约10-20分 | 建议至少输入模型中权重最高的2个可选因子 |
| 全部可选因子缺失 | 仅使用基础因子（PE/PB/PEG/MA/量能/波动率） | 可用，但会丢失行业特异性信息 |

### 5.5 特殊情况

- **bank 和 realestate 模型**：PE权重为0，若所有可选因子缺失，则退化为纯PB+MA+量能+波动率模式，对金融/地产行业精度不足，**强烈建议至少输入ROE和NAV折价**
- **cyclical 模型**：商品价格偏离和产能利用率是周期定位的关键，缺失后将退化为通用模型，**推荐至少输入commodity_dev**
- **所有模型的基础因子（PE/PB/PEG/MA/量能/波动率）始终有效**，不存在缺失问题

## 六、PE/PB 区间设定原则

1. **PE区间应覆盖95%以上的历史数据**：可通过腾讯API获取的PE(TTM)历史数据确定上下界
2. **PB区间一般 0.3-6.0**：金融/资源类偏低（0.3-3.0），科技/消费类偏高（1.5-8.0）
3. **增速假设不宜过高**：建议参考行业研报共识，10-20%为合理区间，周期股可更低
4. **同一公司不同时期参数可能需调整**：建议每半年复核一次区间参数

## 七、评分状态分界线（6级）

| 分数区间 | 估值状态 | 投资含义 |
|----------|----------|----------|
| 80-100 | 极度低估 | 历史性低估区间，具备强烈安全边际 |
| 70-79 | 低估 | 估值偏低，可以考虑分批建仓 |
| 60-69 | 中性偏低 | 估值合理偏下，维持持仓观望 |
| 40-59 | 中性偏高 | 估值合理偏上，关注止盈信号 |
| 20-39 | 高估 | 估值偏高，考虑减仓或观望 |
| 0-19 | 极度高估 | 严重高估，存在较大回调风险 |
